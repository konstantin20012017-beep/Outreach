# Аутрич — стратегические принципы

Skill для этапа «понять кому и зачем пишем» — до того как сесть за конкретный текст цепочки.

## 1. Узкая сегментация

- Не широкие базы, а конкретные сегменты (стоматологии, автосервисы и т.д.).
- Релевантность — главный фактор.

## 2. Персонализация через ценность

- В письмо закладываются 2-3 конкретные идеи под клиента (на основе данных: сайт, описание, гео + AI).
- Каждый получатель видит релевантные именно ему предложения.

## 3. Структура письма: Айсбрекер → Проблема → Решение → CTA

Три формулы (выбирать в зависимости от ниши и наличия данных):
- **Боль → CTA** — базовая.
- **Боль + Доказательство** (цифры, кейс) → CTA.
- **Боль + Частичное решение / Инсайт** (подсказка, от которой клиент задумается) → CTA.

## 4. Айсбрекер на основе триггера

Письмо должно выглядеть так, будто менеджер лично изучил компанию. Типы триггеров:

- **Наличие кейса** — «увидели, что вы в сфере X, у нас есть кейс с вашими коллегами по рынку».
- **Описание компании / сайт** — «побывал на сайте, увидел, что у вас есть [услуга]».
- **Гео** — «мы тоже из [город]».
- **Маркетплейсы** — «увидели вас на маркетплейсах».
- **Вакансии** — сигнал потребности.
- **События / новости** компании.
- **Технологический стек** (CMS, инструменты аналитики).
- **Закон / регуляторный дедлайн** — самый сильный заход, если в нише реально есть такая боль/обязательство («с 1 марта обязали ставить X», «требования идут в регионы», субсидия от государства). НЕ применять, если регуляторной боли в нише нет — не пугать клиентов выдуманными «обязательствами». Использовать только когда в брифе зафиксирован реальный закон/норматив. Если триггер есть — конкретные цифры регуляторики (даты, проценты, мощности, размер субсидии) можно давать прямо в первом письме.

## 5. Если нет явного триггера — квалифицирующий вопрос

- Не утверждать, а спросить: «У вас есть X?» / «Планируете Y?».
- Потом: «Почему спрашиваю — дело в том, что...» + краткий оффер с цифрами.
- Снижает негатив от нерелевантных писем, отсеивает нецелевых.
- **В последнем письме цепочки** — квалифицирующий вопрос особенно уместен: «Если сейчас есть проект в работе / планируется...». Фильтрует кому не актуально, снижает давление.

## 6. Мимикрия под живого менеджера

- Письмо НЕ должно выглядеть как рассылка.
- В некоторых нишах чем проще и «деревяннее», тем лучше.
- Подстраиваться под аудиторию: для «гаражников» — максимально просто, для IT — можно сложнее.
- **Для новых компаний** — информационный тон: «мы запустили / пишу от студии» вместо «я знаю вашу проблему». Задача — донести, что появились на рынке, а не давить.
- **PS-блок «это не AI»** (опционально, для первого письма) — приём для ниш, где аудитория привыкла получать AI-рассылки (IT, digital, маркетинг). Пример: «PS – Понимаю, что сейчас много автоматических AI-рассылок. Это письмо я написал лично и действительно посмотрел ваши работы.»

## 8. Язык аудитории (для твёрдых ниш)

- Писать на языке клиента: использовать фразы, с которыми клиенты обращаются впервые.
- Использовать точные формулировки из звонков и брифов.
- Включать возражения из отдела продаж.
- Конкретика: цифры, бренды, факты.
- Чем короче, твёрже и понятнее — тем лучше.
- «Мы местные, тут работаем» — если применимо.

## Каждая цифра — с привязкой к источнику

Когда в письме появляется конкретная цифра (% эффективности, кратность, срок, сумма) — она атрибутирована, и атрибуция определяет форму подачи:

- **Если цифра есть в кейсе** (PDF «Истории Успеха», пресс-релиз, видеокейс) — атрибутировать к компании: «Билайн — средний чек +24%», «АСКОНА — конверсия x1,5», «ВОЛМА — закупки -20%». Самая сильная форма.
- **Если цифра только в общем брифе клиента** (свод метрик без привязки к одному кейсу) — давать как «в среднем по проектам BPMSoft / агентства»: «маркетинговые расходы – минус 50%». Не приписывать конкретной компании.
- **Если цифра только на лендинге мероприятия / на сайте, в кейсах её нет** — подавать как речь спикера / тезис вебинара: «спикер расскажет, как ускорил БП в 5 раз», «на вебинаре разберём, как повысили эффективность на 30%». Не утверждать как наш факт.

В блоке «Источники» в HTML-цепочке каждая цифра должна иметь точную ссылку: PDF + раздел / пресс-релиз + дата / лендинг + URL / транскрибация + таймкод.

## Сайт клиента — обязательный источник

Клиент в брифе часто упускает свои сильнейшие регалии (узнаваемые бренды, цифры, отзывы). Сайт изучать одновременно с брифом, не «потом».

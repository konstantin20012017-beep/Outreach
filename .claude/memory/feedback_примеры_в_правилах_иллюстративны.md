Когда вношу примеры в skill / фундамент текстов / любой справочник — сразу помечать их как **иллюстративные и не закрытый список**. Не присваивать примеру статус жёсткого правила («использовать только это»). «Как надо»-примеры подавать водянисто и явно писать, что в самом письме формулировки **конкретные под проект из брифа**, не общие слова.

**Why:** за одну сессию (2026-06-04, фундамент текстов) пользователь поправил это трижды подряд: «это просто пример водянистый, под проект конкретно прописывать» / «варианты тем не обязательно как постоянное правило использовать только их» / «тоже варианты». Я по умолчанию подавал примеры как готовый канон, он каждый раз снимал с них статус правила.

**How to apply:**
- Любой банк примеров (темы, опенеры, формулировки) → рядом фраза «не закрытый список, под проект придумывать свои в этой логике».
- «Как надо»-пример → пометка-плашка «иллюстрация принципа; в письме — конкретика из брифа (боль, оффер, действие), не общими словами».
- Не ждать поправки пользователя — ставить пометку сразу при первом внесении.

Связано с правилом skill `аутрич-цепочка` «Присланные примеры — не правила формата» (там про примеры клиента; здесь про МОИ примеры в правилах). См. [[project_фундамент_текстов_синк]].

Если я говорю пользователю «параллельно делаю X, пока ты Y» (или «моя часть / твоя часть»), то X должен запуститься В ТОМ ЖЕ ответе через tool calls. Обещать параллелизм и ничего не делать — нельзя.

**Why:** 2026-05-19 в сессии создания контент-ниши я сказал «Параллельно я пишу промпт Haiku и план рефакторинга» — и стал ждать `chat_id` от пользователя, ничего не делая. Когда пользователь вернулся, он сам поймал это: «ты сказал что параллельно это сделаешь получается халтурил?». Обещание параллельной работы воспринимается как обязательство — невыполнение бьёт по доверию сильнее, чем честное «я жду тебя».

**How to apply:**
- **Триггерные формулировки** для самопроверки: «параллельно», «одновременно», «пока ты делаешь X», «моя часть / твоя часть», «займусь этим, пока...».
- Перед тем как написать любую из этих фраз — спросить себя: «Я СЕЙЧАС в этом сообщении инициирую tool call для X, или просто обещаю?» Если просто обещаю — не писать формулировку, переформулировать.
- **Правильные альтернативы:**
  - «Сейчас начинаю X» + сразу tool call в этом же сообщении.
  - «Я подожду, пока ты сделаешь Y, потом займусь X» — честно, без иллюзии параллелизма.
- **Контр-пример (как НЕ надо):** «Иди создавай канал, я параллельно пишу промпт» — и ни одного tool call в сообщении.
- **Пример (как НАДО):** «Иди создавай канал. Параллельно начинаю промпт.» — и тут же `Write` создающий файл `критерии.md`.

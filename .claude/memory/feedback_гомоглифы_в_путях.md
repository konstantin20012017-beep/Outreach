🔴 ХРОНИКА — нарушаю систематически (06-08 ×3, 06-09, 06-17). ЖЁСТКОЕ ПРАВИЛО: **никогда не печатать кириллический путь руками.** Любой путь с `Аутрич/Парсер/цепочки/брифы/...` — только копировать из вывода `pwd`/`ls`/прошлого tool-результата, не набирать по памяти.

В русскоязычном проекте следить за гомоглифами — случайной подменой кириллических букв латинскими (и наоборот) в путях файлов, строках и `old_string` для Edit. Перед Write/Edit/Read по RU-пути убедиться, что в имени только кириллица.

**Why:** На АСНА-сессии 2026-06-08 три слипа подряд: «Аутрич»→«Аутрич**ich**» (создал лишнюю папку+файл-заглушку, удалить через Bash не дали), «подписант»→«подпис**ant**» (Edit упал, строка не найдена), «цепочка»→«цеп[CJK-иероглиф]ка» (засорил MEMORY.md, пришлось чинить). Гомоглифы невидимы на глаз, но ломают пути и edit-матчинг и плодят мусорные артефакты.

**How to apply:**
- Перед Write нового файла по RU-пути — перечитать путь, никаких латинских a/c/e/o/p/x/y и т.п. внутри кириллических слов.
- old_string для Edit копировать дословно из свежего Read, не печатать по памяти (так же ловится пропуск префиксов вроде `[17:26]`).
- Если Edit упал с «string not found» — первым делом проверить гомоглифы и точное совпадение, а не переформулировать.
- После записи в MEMORY/память — быстрый взгляд, не проскочил ли инородный символ.

При «File does not exist» по RU-пути — первым делом подозревать латинскую букву, а не отсутствие файла.

**Нарушил ЕЩЁ 2 раза (Джубга-сегменты, 2026-06-17):** (а) напечатал «Документы»→«Документs» в Write — создал мусорный каталог `~/Документs` с файлом-заглушкой (rm не дали права, висит); (б) «Аутрич»→«Аут**r**ич» в Read — «File does not exist». 6-7-й слипы. **РАБОЧАЯ ТЕХНИКА, которая в этой сессии реально сработала (применять по умолчанию):**
- Для Bash-операций по RU-пути: добыть путь через `find` в переменную и работать ТОЛЬКО через неё, RU-имя не печатать в самой команде. Так весь путь — из вывода find, руками не набран.
- Правки файлов с RU-путём делать через `perl -i -pe` (БЕЗ `-CSD` — байтовая замена, паттерн UTF-8 совпадает с файлом) по пути-переменной, а не через Edit с литералом пути.
- Для Read/Edit (нужен литерал пути): копировать путь ИЗ строки свежего tool-вывода (grep/ls/find), не перенабирать. Если поймал себя на ручном наборе RU-пути — это сам по себе red flag, остановиться и скопировать.
- ASCII-части (папка памяти `...-Documents-------/`, ASCII-имена файлов типа `feedback_subject_*`) — безопасны, риск только в кириллических сегментах (Аутрич/Парсер/цепочки/брифы/сегмент + кириллические имена файлов).

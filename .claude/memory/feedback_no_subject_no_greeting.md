**Обновление 2026-05-29:** правило проекта изменилось — **тему прописываем в КАЖДОМ письме** цепочки. Пользователь: «у нас обновленное правило – теперь всегда прописываем тему». **Тема ОДИНАКОВАЯ во всех 3 письмах** — чтобы письма склеивались в одну переписку в почтовике (НЕ разная под роль письма). В теме — переменная компании `{Компания}` для персонализации. Тема общая по сегменту, стиль живого менеджера, без пафоса и капса. Клиенту давать 3-4 варианта темы на выбор. Отдельной строкой `Тема: ...` в начале блока, перед `[приветствие]`.

**По-прежнему НЕ пишем в тексте** (только плейсхолдеры):
- Приветствие («Здравствуйте», «Добрый день», `{{доброгоВремениСуток}}`) → `[приветствие]`
- Подпись («С уважением, ...», ФИО менеджера) → `[подпись]`

**Why:** приветствие/подпись вставляются на стороне клиента/системы рассылки. Тема раньше тоже была под запретом (нарушил 2026-05-05 на iSpring), но с 2026-05-29 клиент сделал тему обязательной частью каждого письма.

**How to apply:**
- Pre-flight grep больше НЕ ищет `Тема:` (убрано из справочника `формат-цепочек.md`). Ищет только `Здравствуйте|Добрый день|доброгоВремениСуток|С уважением|^Подпись`.
- Из присланных клиентом примеров по-прежнему берём только содержание; формат (число писем, плейсхолдеры приветствия/подписи) — из CLAUDE.md/skill'а.
- Закреплено: CLAUDE.md красная линия №3, skill `аутрич-цепочка` (раздел «Роли писем»), справочник `формат-цепочек.md`. Связано с [[feedback_examples_arent_rules]].

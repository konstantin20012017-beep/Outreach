Перед тем как предлагать или запускать массовую платную операцию (например прогон Haiku/LLM по сотням сообщений/чатов), **сразу сам прикидывай стоимость в токенах и ₽ и озвучивай её**, не дожидаясь вопроса пользователя.

**Why:** 2026-06-01, бэктест «Контент». Я расписал план с фазой Haiku на сотнях совпадений, но цену не оценил — пользователь сам спросил «а как это по тратам для хайку выйдет?». Только после этого я посчитал (~70-150 ₽). Стоимость — фактор решения «запускать или нет», и пользователь не должен её выпрашивать. Бюджет Haiku у проекта ограничен (~300-800 ₽/мес), разовый прогон может съесть месячный лимит.

**How to apply:**
- Прикидку давать заранее: число вызовов × (токены входа + выхода) × цена модели. Для Haiku 4.5 — ~$1/млн вход, ~$5/млн выход.
- Показывать вилку (ожидаемо / худший случай) и сравнивать с известным бюджетом проекта.
- Если дорого — предлагать более дешёвый путь первым (бесплатный фильтр по ключам/эвристике до платного LLM; дедуп; ограничение выборки). В этой сессии пользователь как раз свернул на «без Haiku, только ключи».
- Платную фазу не запускать без явного «го» — особенно если она необратима по тратам.

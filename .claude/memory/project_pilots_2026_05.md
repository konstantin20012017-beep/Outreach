## Крипто Денис

Контент для ТГ-канала Дениса. Роль пользователя — создавать посты + параллельные задачи.

- **Ритм:** 2 поста / день пн-пт + 1 в сб + 1 в вс = **12 постов / неделя**
- **Понедельник:** еженедельный отчёт (статистика + выполненные задачи + работа над привлечением заявок)
- **Цель мая 2026:** долгосрочный прогрев экспертизы Дениса и его торговой системы
- **Контент-план:** собирается каждое воскресенье на следующую неделю, не вперёд на месяц

## Мой блог

Рилсы → переходы в личный Telegram-канал.

- **Ритм:** ramp-up с 3 до 10 роликов / день за май, плавно по дням (не ступенчато)
- **Бэклог:** 100 офферов → выбрать лучшие → тестировать на охвате в мае
- **Цель:** максимальная конверсия рилса в переход в ТГ-канал

## Как они работают в системе

**Why:** это первая итерация системы планера — два постоянных проекта, на которых проверяется структура «месяц = карточки + еженедельные документы недели + дашборд». После 2-3 недель стабильной работы по тому же шаблону подключаются Ravix, DITEC и другие.

**How to apply:**
- При генерации недельного плана — учитывать ритм каждого (12 постов Денис + дневная норма роликов)
- Метрики (KPI) обновляет пользователь руками раз в неделю в воскресном ревью; не пытаться тянуть из API
- В понедельник утром — собирать отчёт по Крипто Денис (статистика прошлой недели + сделанное)
- Корректировки в течение недели принимать одной командой и править данные в Yonote через API

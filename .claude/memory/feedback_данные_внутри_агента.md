Когда раскладываем систему на агентов: данные не лежат свободными файлами в корне проекта — каждая база живёт **внутри папки агента, который ею владеет**.

**Почему:** это и есть «каждый агент первым делом читает свой источник» (правило 2). Агент + его данные = один узел. Корень не превращается в помойку из разрозненных .md.

**Как применять:** Claude Code сканирует `.claude/agents/` рекурсивно (личность из поля `name`, не из пути). Поэтому структура — папка на агента: `.claude/agents/<агент>/<агент>.md` + владеемые файлы рядом. Пример: библиотекарь владеет `реестр-результатов.md` + `победители/`, кейсолов — `кейсы/`, разведка — `рынок.md`. Чистый процесс (исследователь/копирайтер/ревьюер) своих данных не держит.

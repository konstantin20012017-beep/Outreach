Один запрос = один итоговый файл/документ. Если в задаче есть несколько срезов (чаты+каналы, прошлое+настоящее, плюсы+минусы, разные категории) — собирать их **в один документ с секциями**, а не делать отдельные файлы для каждого среза.

**Why:** параллельные файлы создают трение — пользователю приходится открывать несколько вкладок, мысленно сшивать данные, держать в голове, где что лежит. Один документ = одна сущность. Кейс 2026-05-25: на запрос «вытащи список чатов под нишу Контент» сделал отдельный HTML для чатов, потом отдельный для каналов; пользователь: «объедини все в одном файле, и каналы и чаты».

**How to apply:** перед сохранением артефакта проверить — если в задаче было несколько срезов одной сущности (разные источники того же списка, разные временные окна одного отчёта), собирать в один файл с заголовками-разделами. Разделять файлы только если пользователь явно попросил («сделай отдельно») или если артефакты независимы по сути (бриф и цепочка — разные сущности). Связано с [[feedback_no_silent_structural_drop]] — там про молчаливое удаление секций, здесь про молчаливое разделение.

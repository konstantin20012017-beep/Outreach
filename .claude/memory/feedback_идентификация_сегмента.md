Когда пользователь говорит «первый сегмент / сегмент N / эта цепочка» — не угадывать, какой это файл, по своей внутренней нумерации или «приоритету». Свериться с КАНОНОМ сегментации клиента (целевая должность + боль + триггер). При расхождении между названиями/порядком файлов и тем, что имеет в виду клиент — уточнить ДО правок.

**Why:** 2026-06-24, МояКоманда. Пользователь попросил «исправить первое письмо в первом сегменте». Я принял «первый сегмент» = IT-цепочка (т.к. IT был приоритетом и первым по файлам) и начал её править. Пользователь поправил: «Первый сегмент — это не IT, это HR/HRD, таргет по ДОЛЖНОСТИ». Файлы сегментов назывались по углу (IT/HRD/универсал), а клиентская таблица нумерует по-другому (С1 HR-операционный, С2 HRD). Моя нумерация ≠ нумерация клиента → правка ушла бы не в тот файл.

**How to apply:**
- «Сегмент N» от клиента → открыть канон сегментации (таблица/бриф) и сопоставить по целевой должности/боли, а не по порядку файлов или своему «приоритету».
- Где сегменты названы по углу, а клиент нумерует по таблице — держать маппинг; при сомнении переспросить «это который по должности (HR/HRD) или который по углу (IT)?».
- Если просят «исправить» сегмент, которого ещё нет как файла — сказать прямо, что его нет, и уточнить: создаём новый или правим существующий (а не молча редактировать похожий). Связано с [[feedback_сегмент_не_клон_прошлой_цепочки]].

# Восстановление памяти после клонирования репо

**Как устроено (настроено 2026-06-23):** реальные файлы памяти лежат в репозитории — `Аутрич/.claude/memory/` (версионируются, пушатся на GitHub). А в системной папке Claude (`~/.claude/projects/<sanitized-cwd>/memory`) стоит **симлинк** на эту папку. Claude читает/пишет память по системному пути, но физически она в репо.

**Проблема при восстановлении:** GitHub хранит файлы памяти, но НЕ хранит симлинк (он локальный, вне репо). На новой/чистой машине после `git clone` файлы будут в `.claude/memory/`, но Claude будет искать их по системному пути и не найдёт. Симлинк надо пересоздать вручную — один раз.

**Команда восстановления** (после клонирования репо на новую машину):
```bash
# 1. путь к склонированному репозиторию (поправить, если склонировал в другое место)
REPO="$HOME/Documents/Аутрич"

# 2. системный путь памяти Claude. Имя папки = абсолютный путь к репо,
#    где все не-ASCII и спецсимволы (/, кириллица) заменены на дефисы.
#    Для клона в ~/Documents/Аутрич под пользователем konstantinsabalin это:
MEM="$HOME/.claude/projects/-Users-konstantinsabalin-Documents-------/memory"

# 3. убрать пустую авто-созданную папку (если Claude Code успел её создать) и поставить симлинк
rm -rf "$MEM"
mkdir -p "$(dirname "$MEM")"
ln -s "$REPO/.claude/memory" "$MEM"

# 4. проверка
readlink "$MEM" && [ -f "$MEM/MEMORY.md" ] && echo "OK: память подключена"
```

**Если репо склонирован в другое место / под другим пользователем:** имя папки `<sanitized-cwd>` изменится — оно кодирует НОВЫЙ абсолютный путь к репо (каждый `/` и каждый не-ASCII символ → дефис). Проще всего: запустить любую сессию Claude в склонированной папке, он сам создаст пустую `memory/` по правильному пути — посмотреть какой это путь (`ls -td ~/.claude/projects/*/`), затем выполнить шаг 3 с этим путём.

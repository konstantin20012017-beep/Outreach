При предложении нескольких независимых правок конфига, фильтра, антиключей, настроек — выкладывать **по одному пункту с цитатой-доказательством и ждать решения**, не дампить полный список сразу.

**Why:** пользователь хочет принимать решения осознанно по каждому пункту. На длинный список он либо даст формальное «ок» (что приведёт к перерасширенному фильтру с ложноположительными отсечками), либо попросит сократить — и придётся переделывать подачу. Кейс 2026-05-25: при тюнинге антиключей парсера выложил 4 группы по 5–7 фраз каждая, пользователь ответил «слишком много всего, давай по порядку, каждый пункт отдельно».

**How to apply:** в задачах вида «отсеки лишнее», «настрой фильтр», «расширь конфиг» с несколькими независимыми решениями — показать первый блок (1 группа, 3–6 кандидатов с цитатами из лога/исходника), дождаться выбора, потом следующий. Если задача атомарная (1 правка) — это правило не применяется. См. [[feedback_dont_re_ask]] — оно про обратный кейс (не переспрашивать на согласованное), эти два правила дополняют друг друга.

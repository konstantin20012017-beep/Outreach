Систематизация проекта Аутрич в агентный пайплайн (решено 2026-06-04). Поток проектов идёт нон-стопом — вместо «бриф → пишу по наитию» строим конвейер с библиотекой того, что отработало.

**Два слоя (важно, см. [[feedback_данные_внутри_агента]]):**
- Знание/данные = файлы внутри владеющего агента.
- Процесс = агенты со своим контекстом.
- Skill ≠ агент: skill = методичка в текущий мозг, агент = отдельный мозг. Логика «как писать» остаётся в skill'ах `аутрич-стратегия`/`аутрич-цепочка`, копирайтер их читает, не дублирует.

**6 агентов в `.claude/agents/` (проектные, рекурсивный скан CC):**
1. `аутрич-исследователь` — стадия 1: бриф+исходники → карточка проекта.
2. `аутрич-библиотекарь` — стадия 2: карточка → матч в библиотеке. Владеет `реестр-результатов.md`, `победители/`, `СХЕМА-тегов.md`. «Победитель» = есть реальная метрика в реестре.
3. `аутрич-кейсолов` — citable кейсы (внутр.библиотека → веб). Владеет `кейсы/`.
4. `аутрич-разведка` — на miss библиотекаря + мониторинг рынка. Владеет `рынок.md` (датированный лог, раз в квартал дистилляция в strategy-skill).
5. `аутрич-копирайтер` — стадия 3: пишет, читая skill'ы. Opus.
6. `аутрич-ревьюер` — стадия 4: свежий взгляд, красные линии+PREFLIGHT. Не правит, выносит вердикт. Opus.

Дирижёр = главная сессия + Workflow-скрипт, отдельного агента-оркестратора не плодим.

**Статус: скелет Фазы 0 поднят (2026-06-04).** Реестр засеян бэкфиллом: iso-smk (реальные цифры из `брифы/iso-smk_кейс.md` — готовый прообраз библиотеки победителей), ECS24 (xlsx не разобран), BPMSoft (0 регистраций, антипаттерн со слов).

Когда пользователь сам внёс правку в готовый артефакт (тему письма, подпись, формулировку), эта правка — **сигнал на все версии и следующие цепочки**, а не локальное изменение одного файла.

**Why:** на проекте АСНА (2026-06-05) пользователь в v1 (1) заменил формальную тему «{Компания} х АСНА» на смысловую боль-тему и (2) убрал плейсхолдер `[контакты для ответа]` из подписи. В v2 я обоих не учёл — поставил тему-склейку обратно и вернул `[контакты для ответа]`. Пользователь поймал оба повтора, раздражён: «ты почему-то продолжаешь это делать».

**How to apply:**
- Перед генерацией новой версии — **сверить с последней правленной пользователем версией** (diff/чтение): что он поменял руками = новый дефолт.
- Не возвращать удалённые им элементы (плейсхолдеры, формулировки) «потому что так в стандарте».
- Темы писем: «{Компания} х Бренд» — слабый дефолт, не цепляет (читается как рассылка). Тема = смысловая зацепка под боль/интригу аудитории. Давать клиенту несколько вариантов на выбор.
- **Тема-вердикт сильнее темы-вопроса.** На АСНА пользователь попросил в заголовке «конкретный драматичный вердикт, не вопрос» (выбрал «{Компания}: 2026 выдавливает с рынка малые аптеки»). Дефолт для тем — утверждение-вердикт под подтверждённую источником боль, а не вопрос. ⚠ Но красная линия #2: конкретную цифру-прогноз («N% закроется») не выдумывать — драматизировать можно только тезис, который уже есть в источнике (лендинг/бриф).
- **Слабая тема = generic-пересказ продукта.** «За счёт чего растут независимые сети», «Как небольшой сети зарабатывать больше» — описательные, без крючка, пользователь бракует («заголовок слабый»). Давать варианты **сгруппированными по разным крючкам** (любопытство / личный вопрос ЛПР / деньги-доходность / голос отправителя от первого лица / событие+дата), а не вариации одной формулировки. На АСНА 2026-06-05 сработала только подача темы через таблицу «крючок → тема», из которой пользователь выбрал событийную («Первые лица аптечного бизнеса, 23 июня»). Короткая тема-анонс события часто сильнее описательной.

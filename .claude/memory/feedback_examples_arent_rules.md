Когда клиент присылает свои примеры цепочек/писем — это референс по содержанию (тон, факты, спикеры, цифры), а не правила формата.

Конкретно нельзя выводить из присланных примеров:
- Число писем в цепочке (всегда 3 по правилу проекта).
- Наличие или отсутствие темы (никогда не пишем).
- Формы приветствия и подписи (всегда `[приветствие]` / `[подпись]`).
- Переменные клиента типа `{{доброгоВремениСуток}}`, `{Компания}` — могут быть техническими шаблонами для их рассылочной системы, а в финальный текст идут только переменные из правил проекта.

**Why:** 2026-05-05 на цепочке Кормишин/iSpring я дважды подряд переписал формат из присланных черновиков (сначала «2 письма не 3», потом «тема и приветствие пишутся»). Оба раза пользователь поправил. Корневая причина — я подменял правила проекта примерами клиента, не сверяясь с CLAUDE.md.

**How to apply:**
- Из примеров клиента берём только содержание: спикеров, факты, цифры, цитаты, тон обращения, структуру содержательных блоков.
- Формат (число писем, тема, приветствие, подпись, переменные) всегда из CLAUDE.md.
- Если в примерах формат отличается от правил проекта — это не «спецформат клиента», а либо черновик, либо техника другой системы рассылок. Не копировать.
- Перед отдачей цепочки прогонять чеклист из CLAUDE.md (грепы на запрещённые паттерны).

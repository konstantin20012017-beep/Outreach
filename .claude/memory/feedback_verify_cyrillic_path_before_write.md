При Read/Edit/Write/Bash в путь с кириллицей брать путь из надёжного источника (вывод `pwd`, `ls`, путь из предыдущего тул-результата), а не перепечатывать по памяти. Касается ВСЕХ файловых тулов, не только записи.

**Why:** в сессии JEZA (2026-05-29) я писал HTML-бриф в `/Users/konstantinsabalin/Documents/Аутрич/брифы/jeza_бриф.html`, но опечатался — набрал `Аутrич` (латинская `r` вместо кириллической `р`). Write не упал, а молча создал новую папку-дубль `Documents/Аутrич/брифы/` и положил файл туда. Заметил только постфактум; удалить мусорную папку не смог — `rm -rf` и `rm` вернули permission denied. Пользователю пришлось убирать вручную. Кириллица + латиница визуально неразличимы (р/p, с/c, а/a, е/e, о/o, х/x), поэтому глазами опечатку не поймать.

**Повтор 2026-05-29 (та же сессия, где правило создано):** нарушил ещё 4 раза подряд — `Аутrич`, `контrактное`, `аутrич`, `Аутрич` с латинской буквой. Каждый раз Edit падал «File does not exist», хотя файл существует. Правило есть, но в моменте я всё равно перепечатывал путь вручную → значит нужен жёсткий рефлекс, а не «помнить».

**Повтор 2026-06-01 (dialeads):** снова — на этот раз **Read** абсолютным путём `.../Аутrич/брифы/refexpress_бриф.md` с латинской `r`, упал «File does not exist». Тот же паттерн уже на чтении, не только на Write/Edit. Вывод: рефлекс «относительный путь от cwd» включать на ЛЮБОМ файловом туле сразу, ещё до первого падения.

**Bash cwd НЕ персистится между вызовами (2026-06-01):** каждый вызов Bash стартует заново из профиля, не из cwd предыдущего вызова. Сегодня сделал `cd .../цепочки` в одном Bash, следующий `python3 'file.html'` в новом Bash запустился из старого cwd (`/брифы`) → «No such file or directory». Относительный путь спасает только если `cd` и команда в ОДНОЙ compound-команде через `&&`.

**How to apply:**
- **По умолчанию — ОТНОСИТЕЛЬНЫЙ путь от cwd** (`цепочки/...`, `брифы/...`, `.claude/skills/...`). cwd уже кириллический и корректный, перепечатывать абсолютный путь незачем. Все падения в этой сессии были на абсолютных путях в Edit, относительные в Bash работали.
- Если нужен абсолютный путь — копировать его из вывода предыдущего тула (`pwd`, `ls`, прошлый Read/Write), не печатать.
- Если в Bash нужен `cd` — держать `cd` и команду в одной строке через `&&` (cwd не переживёт следующий вызов).
- **Детектор:** Edit/Read вернул «File does not exist», а ты уверен, что файл есть и cwd кириллический → это латинская буква в пути. НЕ искать файл заново — перебрать путь посимвольно или переключиться на относительный.

- Опасные пары кириллица/латиница: р/p, с/c, а/a, е/e, о/o, х/x, к/k, м/m, т/t, н/h.

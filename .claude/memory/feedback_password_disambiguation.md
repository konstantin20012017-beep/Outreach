Когда пользователь присылает в чат строку, похожую на пароль, **без указания** «это для X», нужно одной фразой переспросить, к чему этот пароль относится: VPS root / 2FA Telegram (облачный пароль) / Anthropic API / TG-бот / что-то другое.

**Why:** в сессии 2026-05-20 пользователь скинул пароль `kN7dS0bT1b` без подписи. На предыдущих шагах обсуждался QR-логин Telethon (`login_vps.py`) с запросом 2FA → я принял пароль за 2FA-пароль и потратил время на попытки логина. Пользователь поправил: «я дал тебе пароль в предыдущем сообщении…» — это был **новый root-пароль VPS** после ротации. Угадывание из ближайшего контекста = разрушение времени и неверный диагноз.

**How to apply:**
- Увидел строку без метки, по форме похожую на пароль → одна короткая фраза: «Спасибо. Это для X (VPS root) или Y (2FA Telegram)?»
- Не угадывать по последнему обсуждённому артефакту. Контекст подсказывает, но не доказывает.
- Связано: [[feedback-parser-vps-no-password]] — root-пароль VPS обычно агенту не даётся, поэтому появление пароля = редкое событие, важно правильно его атрибутировать.

Когда нужно отдать пользователю набор команд для терминала (деплой, проверка логов, разовый SSH-сценарий, любое многошаговое действие на сервере) — сразу делать отдельный HTML-файл в директории проекта с кнопкой «Копировать» на КАЖДОЙ команде. Открывать через `open -a "Google Chrome"`.

**Why:** пользователь работает только из терминала, копировать команды из markdown-блока в чате неудобно. HTML с `navigator.clipboard.writeText()` гарантирует идеальный paste. За сессии формат запрашивался многократно — устоявшийся, не разовая просьба.

**Усиление (2026-05-20):** пользователь повторно подчеркнул — команды в инлайн-блоке чата он копировать **не может**. Никогда не отдавать команды inline (даже одну короткую), всегда HTML в отдельной вкладке. Если кажется, что «одна простая команда — можно без HTML», всё равно делать HTML.

**Дополнительно (2026-05-19):** при рефакторинге мультиниши я сделал кнопки `position: absolute; top: 8px; right: 8px;` поверх блока `<pre>`. У пользователя они отображались неправильно (либо все наложились друг на друга, либо ушли из видимости). Пользователь явно сказал: «У меня нет возможности копировать конкретную команду под конкретным пунктом. Сделаю возможность, чтобы можно было у каждой конкретной команды сделать копию этой каждой конкретной команды.» Поэтому кнопки делать **обычными блочными элементами под командой**, без хитрого позиционирования.

**How to apply:**
- Триггер: я собираюсь выдать в чате 2+ команды подряд, или одну длинную команду с кавычками/awk/grep.
- Делать HTML самому, не ждать просьбы. После создания — открыть в Chrome.
- Структура каждой команды:
  ```html
  <div class="cmd-box" id="cmdN">...команда...</div>
  <button class="copy-btn" data-target="cmdN">Копировать</button>
  ```
  Кнопка — `display: block; margin-top: 8-10px;` под блоком. Заметная (зелёный фон, белый/чёрный текст), не серая.
- **НЕ делать** `position: absolute` для кнопок — они теряются, накладываются, не видны при overflow-x scroll.
- Перед отдачей пользователю — мысленно проверить: для каждой команды кнопка видна и явно отделена визуально.
- Тёмная тема, моноширный шрифт в командах. Файл называть осмысленно (`деплой_чаты_DD_MM.html`, `доделка_приватных_чатов.html`).

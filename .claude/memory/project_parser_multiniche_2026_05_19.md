# Парсер: мультиниши + ниша «Контент» (2026-05-19)

Парсер на VPS работает в режиме нескольких ниш. Конфиг каждой — в папке `ниши/<slug>/` (niche.json + чаты.txt + ключи.txt + антиключи.txt + haiku_prompt.txt). Движок один на все ниши, промпт Haiku — свой у каждой.

**Активные ниши (51 чат, 2 ниши):**
- 🚗 Авто из Китая — 43 чата, 356 ключей, 44 антикл. Выход: канал «Авто из Китая – лиды» по invite (как раньше). Логов в TG пока нет.
- ✍️ Контент — 12 чатов (8 + 4 инфобиз-вакансийных от 2026-06-02), ~18 ключей-ролей, ~50 антикл. Услуга «отдел контента под ключ». Выход: супергруппа «Парсер — Контент» (`-1003963298815`), топик «Лиды» (3) и топик «Логи» (5). Haiku пишет ВСЕ вердикты (одобрено + отклонено с причиной) в «Логи», одобренные с пометкой 🔥 если онлайн-школа — в «Лиды».

**Сужение ниши «Контент» 2026-06-02 (важно):** по решению пользователя ДА (лид) теперь ТОЛЬКО если клиент — онлайн-школа / инфобиз / эксперт-наставник / онлайн-курс / edtech / личный бренд эксперта. Раньше любая ниша проходила как «ОБЫЧНЫЙ» — теперь весь **офлайн-бизнес и любой не-образовательный онлайн** (товарные бренды, FMCG, ритейл, общепит, салоны, фитнес, клиники, недвижка, юристы, авто, производство, performance-агентства, IT) и **ЭЗОТЕРИКА** (таро, астрология, нумерология, матрица судьбы, дизайн человека, магия, энергопрактики, расстановки) — в отказ. Психология/коучинг/финансы/здоровье/языки/маркетинг — НЕ эзотерика, берём. Зашито в `ниши/контент/haiku_prompt.txt` (структура «ГЛАВНОЕ ПРАВИЛО НИШИ» + 3 обязательных условия для ДА). Промпт проверен на 11 реальных кейсах из логов (11/11). Поток станет меньше, но релевантнее.

**Why:** пользователь хотел видеть работу Haiku в TG (без SSH), и добавить новую нишу без пересборки кода. Архитектура «общий движок + папка на нишу» — масштабируется на любое число ниш.

**How to apply:** для новой ниши — создать `ниши/<slug>/` с 5 файлами, перезалить + рестарт. Код трогать не нужно. См. структуру в существующих нишах. Бот `@avto_leads_bot` общий, добавляется админом в каждый канал/группу. Приватные TG-чаты в `чаты.txt` через invite-ссылку `https://t.me/+HASH` — резолвятся через `CheckChatInviteRequest`.

⚠️ **При правках чатов/ключей/антиключей/промпта — ТОЛЬКО файлы в `ниши/<slug>/`.** Корневые `авто_китай_*.txt` / `контент_*.txt` в `/opt/parser/` и локально — это legacy-бэкапы, парсер их НЕ читает. 2026-05-22 я отредактировал корневой `авто_китай_чаты.txt`, задеплоил, парсер показал «43 чатов» вместо 47 — пришлось повторять через `ниши/авто_китай/чаты.txt`. Перед правкой — `ls ниши/<slug>/`, не наоборот.

**Открытые вопросы / отложено:**
- Авто-ниша без логов в TG — пользователь добавит супергруппу с топиками позже, когда захочет.
- Старый канал «Контент – лиды» (`-1003906967474`) — не используется, можно удалить.
- Старые `.txt` в корне `/opt/parser/` на VPS — оставлены как бэкап, удалить через 1-2 дня успеха.
- Пароль root от VPS засветился в чате 2026-05-19 — пользователь сменит сам.
- Inline-корректировка промпта через ответы в TG — идея на потом, не делаем.

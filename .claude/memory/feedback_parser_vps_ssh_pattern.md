При диагностике/правке парсера на VPS `85.137.165.185`:

1. **Первая команда — только заход:** `ssh root@85.137.165.185`. Пароль агент НЕ знает и НЕ вставляет — пользователь вводит сам в терминале.
2. **Дальнейшие команды — без `sshpass`, без обёртки в `ssh ... "..."`** — это команды, которые выполняются уже внутри открытой SSH-сессии (`tail`, `grep`, `systemctl`, `cat`).
3. Каждая команда — в своём блоке с отдельной кнопкой копирования (правило `feedback_commands_as_html_with_copy`).

**Why:** Пользователь уже несколько раз правил агента, который пытался зашить пароль в команды (`sshpass -p '...' ssh ...`). Пароль агенту не показывается ([[feedback_parser_vps_no_password]]), и любая команда с паролем неверна. Плюс пользователю удобнее работать в одной SSH-сессии, а не открывать соединение под каждую команду.

**How to apply:** При любой диагностике/деплое на VPS парсера — структура HTML:
- Блок 1: `ssh root@85.137.165.185`
- Блок 2..N: команды для выполнения внутри сессии (`tail -100 /var/log/parser.log`, `cat /opt/parser/ниши/.../чаты.txt`, `systemctl status parser`).

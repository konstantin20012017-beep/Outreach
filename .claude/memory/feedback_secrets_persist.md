Когда пользователь присылает токен / API-ключ / ID воркспейса / любой рабочий реквизит — сразу сохранять в `.env` (или аналогичный файл) в папке проекта. Не полагаться на то, что значение «останется в контексте» — между сессиями контекст теряется, и пользователю приходится присылать повторно.

**Why:** в сессии 2026-05-03 пользователь прислал токен Yonote, я не сохранил его в файл. На следующей сессии токена не оказалось, пользователь справедливо разозлился: «я тебе в конце прошлой сессии специально написал, чтобы ты все важное зафиксировал».

**How to apply:** как только пользователь присылает любой секрет / реквизит — в том же ответе писать его в файл (`.env`, `config.json` и т.п.) с правами 600, и упоминать в резюме беседы путь до файла. Если пользователь в конце сессии говорит «зафиксируй важное» — в первую очередь проверять, что секреты лежат на диске, а не только в тексте резюме.

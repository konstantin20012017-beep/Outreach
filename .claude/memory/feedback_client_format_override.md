Когда клиент явно просит повторить нестандартный формат цепочки — следуем просьбе, даже если в брифе/CLAUDE.md зафиксирован другой стандарт. Исключение фиксируем на уровне сегмента (в брифе клиента + в memory project), а не как новое общее правило.

**Why:** Стандарт проекта (3 письма, плейсхолдеры `[приветствие]`/`[подпись]`) — это дефолт для большинства случаев. Но у клиентов бывают свои устоявшиеся форматы (например, у Кормишина для Победы 26 мая — 2 письма с темой и `{{доброгоВремениСуток}}`, повтор тестовых рассылок от 02.04). Прямая просьба клиента про конкретный сегмент важнее зафиксированного дефолта. При этом нельзя автоматически распространять исключение на остальные сегменты — другие цепочки того же клиента могут оставаться по стандарту.

**How to apply:**
- Если есть противоречие между правилами проекта и просьбой клиента — задать один уточняющий вопрос («формат по проекту или по присланным примерам?»), затем следовать ответу.
- Зафиксировать выбор в `брифы/<клиент>_бриф.md/.html` как явное исключение для конкретного сегмента, а не как новое общее правило.
- В warning-card HTML-цепочки явно указать «формат отличается от стандарта проекта по прямой просьбе клиента, применять только к сегменту X».
- НЕ обновлять CLAUDE.md под одноразовое исключение.
- Аналог: ravix-style.md — там тоже клиентский стиль (Natural Writing) задан на уровне клиента, а не глобально.

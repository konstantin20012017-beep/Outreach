Когда открываю несколько HTML-файлов в Chrome подряд, фокус достаётся последнему открытому. Главный/только что созданный артефакт нужно открывать последним, иначе юзер видит не то, что ждал.

**Why:** в сессии dialeads (2026-06-01) после написания цепочки С3 я открыл одной командой сначала цепочку С3, потом бриф — в фокус попал бриф. Юзер: «ты мне только бриф открыл, открой файл с третьей цепочкой». Он ждал на экране новый рабочий артефакт (цепочку), а получил вторичный (бриф).

**How to apply:**
- Если юзер просит открыть **конкретный** файл — открывать **только его**, без довеска.
- Если открываю несколько (например, бриф + цепочку после правок) — целевой/новый артефакт открывать **последним**, чтобы он оказался в фокусе. Порядок `open брифа` → `open цепочки`, не наоборот.
- При сомнении, что важнее показать, — открыть один главный, а про остальные сказать словами «бриф тоже обновил, путь такой-то».

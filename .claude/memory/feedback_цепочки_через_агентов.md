В проекте Аутрич готов пайплайн из 6 агентов (`.claude/agents/`). Любая работа над цепочкой — написание, правка версии, доводка темы/подписи — идёт **через агентов**, а не через прямой Write оркестратором.

**Why:** на АСНА (2026-06-05) цепочку v1 я написал руками сам, хотя пайплайн (копирайтер стадия 3 → ревьюер стадия 4) существовал и описан в [[project_пайплайн_аутрича]]. Пользователь поймал: «не вижу что ты используешь агентов, опять сам всё вручную делаешь?». Это прямое нарушение правила #1 глобального CLAUDE.md («дроби на роли») и правила #4 («контроль качества в два слоя») — ручная работа = один мозг без второго слоя ревью.

**How to apply:**
- **Старт любой цепочки = запуск `аутрич-копирайтер`** (Agent tool), не Write своими руками. Оркестратор даёт контекст и ограничения, копирайтер пишет.
- После копирайтера — обязательно `аутрич-ревьюер` свежим взглядом (второй слой, правило #4). Сам себя копирайтер не утверждает.
- Правки готовой цепочки (тема, подпись, формулировка) — тоже через копирайтера, не правь файл сам. Исключение — только если пользователь явно просит «поправь сам по-быстрому».
- **Баланс со скоростью (BPMSoft 2026-06-05, юзер дважды «долго, ускоряйся», прервал ревьюера 2×):** полный пайплайн копирайтер→ревьюер — на СОЗДАНИЕ цепочки и крупную переработку (новый угол, переписка тона всех писем). На МЕЛКИЕ итеративные правки (одна тема, один опенер, убрать строку) — оркестратор вписывает сам и держит гейт (grep + source-check), не гоняет 2 агентов на каждый твик. Если копирайтер уже прошёл чеклист в этой итерации — ревьюера второй раз не запускать ради мелочи. Не превращать «дробить на роли» в ритуал, который злит пользователя медлительностью.
- Оркестратор: собирает бриф/исходники, ставит ТЗ агентам, сводит вердикты, общается с пользователем. НЕ пишет тело писем руками.
- Связано с [[feedback_оркестратор_проверяет_гейт]] (оркестратор ещё и проверяет, что агенты прогнали весь гейт).

**Нарушил снова (Re:activno, 2026-06-09):** несмотря на speed-note выше, запустил полный субагент `аутрич-копирайтер` на правку ОДНОГО письма (доп-жим в П3) — ~8 мин. Пользователь прервал tool use + «очень долго, 8 минут, ускоряйся». Жёстко: правка 1 письма / 1 темы / 1 строки / 1 опенера = оркестратор делает Edit САМ + держит гейт (grep + source-check), субагента НЕ звать. Субагент-копирайтер — только на СОЗДАНИЕ цепочки или крупную переработку. Прикидывать время: если правка занимает у меня 1 Edit — это не работа для 8-минутного агента.

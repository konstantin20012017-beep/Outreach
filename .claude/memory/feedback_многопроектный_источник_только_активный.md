Если входящий материал (голосовое, документ, презентация) накрывает НЕСКОЛЬКО проектов/клиентов — заводить бриф/исходники и работать только по АКТИВНОМУ проекту. Не создавать файлы по проектам, которые упомянуты вскользь / для контекста, пока пользователь явно не попросит их вести.

**Why:** 2026-06-24, Lexis Voice. Голосовое Кости накрывало 2 клиента — Lexis Voice (активный) и Numis (новый, упомянут). Я по красной линии «всё входящее → в бриф» завёл `numis_бриф.md` + `numis_исходники.md`. Пользователь прервал: «не надо пока нумис, я же сказал, бери инфу только для действующего проекта, lexis voice». Сам Костя в голосовом тоже сказал «возьми инфу только по нашему проекту» — я пропустил этот фильтр и расплодил файлы по неактивному клиенту.

**How to apply:**
- Перед заведением нового проекта из многопроектного источника — спросить себя: это активный проект или фоновое упоминание? Если фон — НЕ заводить, максимум одна строка-заметка «источник также касался X, вне scope».
- Это НЕ отменяет CLAUDE №5 («любое входящее → в бриф»), а сужает: «в бриф» = в бриф АКТИВНОГО проекта. Нюанс к красной линии, не противоречие.
- Если в самом источнике автор говорит «бери только по нашему/этому проекту» — это явный фильтр, выполнять буквально.
- Связано с [[feedback_первоисточник_не_выжимка]].

Состав подписи письма должен соответствовать выбранной механике CTA. Если CTA = «ответьте на это письмо» (двухшаговое закрытие), то **блок контактов в подписи лишний** — ответ и так уходит на адрес отправителя, а лишний телефон/сайт читается как «продажник» и бьёт по мимикрии под живого человека.

**Why:** сессия 2026-06-05, цепочка АСНА (`аптека_сегмент1_v1.html`). Заложил в подпись плейсхолдер `[контакты для ответа]` по инерции «у живого менеджера есть контакты», хотя CTA во всех 3 письмах — «ответьте на письмо, пришлю ссылку». Пользователь споткнулся: «почему тут этот блок?». Убрали — подпись осталась чистой (имя + должность + регалии).

**How to apply:**
- Перед тем как класть блок контактов в подпись — свериться с CTA. CTA «ответьте на письмо» → контакты не нужны. CTA «позвоните/напишите в ТГ» → контакты обязательны.
- Когда контакты не нужны — подпись = имя + должность + регалии, без телефона/сайта.
- Это убирает и ложный «блокер»: не запрашивать у клиента контакты для подписи, если механика их не требует.
- Связано с двухшаговым CTA в skill `аутрич-цепочка` (CTA-лестница) и [[feedback_no_subject_no_greeting]].

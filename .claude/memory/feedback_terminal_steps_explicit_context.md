В пошаговых инструкциях для терминала **по умолчанию все команды должны выполняться из одного окна** на маке. Серверные операции — через `sshpass ssh ... 'cmd'`-обёртку. Никаких бэйджей «MAC vs VPS», никаких переключений контекста — пользователь идёт по списку и копирует подряд.

**Why:**
- **2026-05-19 (первый раз):** при деплое 11 чатов пользователь скопировал `scp` и запустил внутри ssh-сессии (без явного контекста — `No such file or directory`). Тогда я решил: добавить MAC/VPS бэйджи.
- **2026-05-19 (вечером, рефакторинг мультиниши):** я применил бэйджи MAC/VPS как правило диктовало. Пользователь явно сказал: «вот эти разделения, они мне ни о чём не говорят. Просто скажи мне, правильно понимаю, что я просто иду по порядку. Первую команду скопировал, ввел, вставил, и вторую команду скопировал, вставил и так далее до конца, правильно?»
- Вывод: бэйджи не решают проблему, они её перекладывают на пользователя («думай, в каком ты сейчас контексте»). Правильное решение — техническое: обернуть серверные команды так, чтобы они тоже запускались с мака.

**How to apply:**
- **Дефолт:** все команды в HTML — для одного окна на маке. Серверные — через `sshpass -p '<pass>' ssh -o StrictHostKeyChecking=no root@<ip> 'systemctl restart ... && tail -n 60 /var/log/...'`. Никаких «теперь подключись к серверу».
- **Когда нужны бэйджи MAC/VPS** (исключение, не правило): если серверная операция интерактивная (например, редактор, или нужно много команд подряд с локальным окружением сервера), и оборачивать в одну строку нельзя. Тогда — да, явные бэйджи с легендой по приглашению (`konstantin@... %` vs `root@server:~#`).
- **Линейный порядок:** пользователь идёт сверху вниз, не думая «а в каком я контексте». В HTML — никаких условных ветвей, развилок, «если ты ещё в ssh».
- Связано с [[feedback-commands-as-html-with-copy]] (формат подачи) и [[feedback-parser-vps-no-password]] (пароль агенту не давать, обёртка строится с плейсхолдером `<root_pass>`).

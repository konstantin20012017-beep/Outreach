## Контекст и боль

У пользователя ~27 задач в бэклоге + 2 активных проекта (Ravix, DITEC) + 2-3 входящих + личный блог. Текущий ежедневник в Yonote — плоский markdown-чеклист в коллекции «Месяцы» (документ на неделю).

**Главные боли:**
1. Бэклог не всплывает — лежит отдельным списком, задачи забываются
2. Каждое утро тратится энергия на ручной обход проектов и сборку задач на день
3. Нет видимого прогресса по проектам
4. Хочется раз в неделю вкладываться в ревью и получать готовый план на след. неделю — без ручной операционки

## Что выбрали (после обсуждения альтернатив)

- **Yonote оставляем** — священная корова, Notion не нравится визуально (громоздкий)
- **Бесплатно**, личное использование, максимум 2-3 человека команды в будущем
- **Дашборд важнее ввода/напоминаний** на первом шаге
- **Идти постепенно**, без большого переустройства
- **Операционку (заполнение, переносы) делает Клод**, не пользователь

## План шагов

1. **Создать новую коллекцию «Планер»** в Yonote — отдельно, чтобы не ломать существующую коллекцию «Месяцы» (старое = архив)
2. Внутри — БД-страница «Задачи» с полями: Проект, Статус, Срок, Повтор (нет/еженед/ежемес/дата), Время факт, Доход
3. Пилот: распарсить документ «20 - 26 апреля» → залить задачи в БД с проставленными проектами → показать пользователю
4. Если ок — залить остальные недели + бэклог-27
5. Воскресный ритуал (через 2 недели после приживания структуры): Клод через MCP/API смотрит Yonote → отчёт за прошлую неделю + предложение плана на следующую → пользователь утверждает
6. HTML-дашборд (Chart.js, локальный файл) — третий шаг, после структуры

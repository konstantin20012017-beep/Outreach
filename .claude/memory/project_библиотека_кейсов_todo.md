**Кто будет искать:** либо сам пишущий агент (`аутрич-копирайтер`), либо — что пользователь считает лучше — отдельный агент, который подбирает структуру/кейс. Задача такого агента: по новому проекту найти конкретный нужный кейс и подходящую структуру (каркас цепочки).

**Что уже есть (достраивать, не с нуля):**
- `аутрич-кейсолов` — агент, владеет папкой `.claude/agents/аутрич-кейсолов/кейсы/` (пока README). Стадия 2.5 пайплайна — находит релевантные кейсы с цифрами.
- `аутрич-библиотекарь` — владеет `.claude/agents/аутрич-библиотекарь/победители/` + реестром результатов. Стадия 2 — ищет отработавший каркас в нише/смежной.

Идея пользователя по сути = наполнить эти библиотеки реальными кейсами и довести поиск «кейс+структура под новый проект» до рабочего состояния. Связано с [[project_пайплайн_аутрича]].

# Библиотека победителей

Сюда кладём **только отработавшие** цепочки и выжимку «что сработало» по нишам. Не склад всех черновиков (это папка `цепочки/`), а полка с наградами.

## Что такое запись победителя
Один файл на нишу или на устойчивый паттерн. Внутри:
- Метрика из реестра (reply/lead %).
- **Почему сработало** — конкретный механизм (триггер, структура, формулы).
- Структуры писем №1/№2/№3 как референс (содержание, не дословный текст).
- **Антипаттерны** — что в этой же нише отработало хуже и почему. Уроки ценнее побед.
- Теги по `СХЕМА-тегов.md`.

## Как сюда попадают
1. Кампания отправлена → цифры в `реестр-результатов.md`.
2. Цифры выше базовой по нише → библиотекарь оформляет запись победителя.
3. Слабая кампания → фиксируем как антипаттерн (отдельной заметкой или внутри записи ниши).

## Зарегистрированные победители
- **Сертификация/СРО** → `брифы/iso-smk_кейс.md` (внешний референс, не дублируем — ссылаемся). Метрики сведены в реестр.

> По мере возврата данных от клиентов сюда переезжают подтверждённые ECS24, JEZA, Kormishin и др. Файл `iso-smk_кейс.md` со временем стоит перенести физически в эту папку — пока оставлен на месте, чтобы не порвать ссылки из ecs24-работ.

Re:activno – системный маркетинг
Сегмент 2: Триггер «вакансии» (аутсорс-команда наряду с наймом) – v1

Угол цепочки. Триггер – у компании открыта вакансия (SEO-специалист / маркетолог / руководитель маркетинга). Сигнал: компания уже решила усилить маркетинг и предпринимает действие (нанимает). Заходим с этого факта и предлагаем посмотреть на ту же задачу через команду-подрядчика наряду с наймом одного штатного. Уровень Ханта: по «нужен маркетинг» – Solution Aware (уже нанимают), по «инхаус-наем долго/дорого/узок» – Problem Aware, отсюда главный аргумент. Ключевой УТП-стержень всей цепочки – насмотренность: через агентство проходит больше проектов и ниш, чем видит один штатный спец. SEO в письме не продаем – закрываем на ответ «интересно» (двухшаговый CTA: ответ → менеджер берет телефон → ценность на звонке). №1 – легкий контакт (вакансия + идея аутсорса + мягкий вопрос), №2 – почему команда сильнее одного найма (насмотренность, состав команды по цене одного, кейс по нише) + оффер аудита, №3 – короткий доп-жим новым аргументом (скорость старта без долгого поиска и онбординга) + NDA для промышленных. Тема одна на все 3 письма, несет триггер найма/маркетинга.
Плейсхолдеры и открытые поля – заполнить до отправки:

1. Вакансия (роль). В тексте плейсхолдер [вакансия – какую роль ищут, подставит оператор] – оператор подставляет реальную роль из конкретной вакансии получателя (SEO-специалист / маркетолог / руководитель маркетинга). Конкретику вакансии не выдумываем.

2. Кейс под нишу получателя. В П2 для примера стоит кейс из медицины (медицинская одежда, B2B) – одна из целевых отраслей сегмента. Под фактическую нишу получателя оператор подставляет релевантный кейс из Источника 8 (например для услуг для бизнеса / клиник – соответствующий кейс с цифрами). Цифры – строго из брифы/reactivno_исходники.md, Источник 8. Год не указываем (просьба клиента). Имя клиента-кейса – по согласованию (на сайте кейсы анонимны по нише).

3. Подписант и контакт для ответа. От кого идет рассылка и куда уходит «интересно» – не задано клиентом. В письмах плейсхолдеры [приветствие] и [подпись], в подписи – [контакт для ответа – от кого идет рассылка, заполнит клиент].

4. NDA (П3). Упоминание NDA подается как опция для промышленных / производственных получателей (бриф: «NDA – важно для заводов»). Если получатель не из промышленности – оператор может убрать строку про NDA.

Умышленно опущено: конкретные сроки найма/онбординга и зарплата штатного спеца (источника нет – в тексте качественно, без чисел). Возражение «гарантии» – по решению клиента не отрабатываем и не упоминаем.

## ЛПР — уточнено

- Основной — **собственник / директор** (не зависит от тарифа).
- **РОП — сильное влияние + риск саботажа** («мне навязывают — саботирую»). → **пишем собственнику**, либо делаем РОПа союзником, а не врагом. В тексте писем не подставлять РОПа («ваш РОП не справляется») — это путь к саботажу.
- Что важно ЛПР: быстрая практическая польза, простота внедрения без IT, понятность, применимость каждый день, скорость инсайтов. **Цена вторична**, если чувствуют «это даст управление».

## Сегментация цепочек — 3 ТАКТИКИ (утверждено как направление)

Стратег задал три типа аутрича. Это и есть оси, по которым делаем разные цепочки:

| # | Тактика | Кому / по чему собираем базу | Заход письма |
|-|-|-|-|
| **1. ICP + кейс по нише** | Базовый, массовее | По ОКВЭД/географии/обороту внутри приоритетной ниши (опт, оборудование, производство…) | «Видим, вы в сфере X → у нас есть кейс по вашей нише → проблема → результат» |
| **2. Триггер — вакансии** | Тёплее, точечно | Кто **активно набирает менеджеров по продажам** (сигнал роста + наличие ОП). Скрещивать с отраслью/оборотом от 200 млн | «Видим, расширяете отдел продаж → растущей команде сложно удержать управляемость → …» |
| **3. ЛПР по должности** | Точечно, по резюме-базам | Должности: **РОП** (основной), реже директор/комдир/собственник | Прямое письмо ЛПР на боль управляемости |

> Под каждую нишу — отдельная цепочка, ниши «штампуем». База собирается **индивидуально под нишу, малыми списками 500-1500 контактов** (не массово). Первую цепочку делаем под максимальный приоритет (см. дополнение выше).

## 🔑 Общая почта vs ЛПР — влияет на заход письма

- В РФ **общие почты** в среднем/крупном бизнесе работают **лучше** ЛПР-почт (в крупном почты устаревают, ЛПР перегружены, секретари фильтруют). ЛПР-почты — для малого бизнеса.
- **Заход на общую почту НЕ через прямую боль** («руководители не могут принять решение») — секретарь читает как рекламу.
- **Рабочий заход на общую почту: «у нас релевантный кейс по вашей нише, ищем контакт нужного коллеги — подскажите/перешлите»**. Это проходит фильтр секретаря.
- → Если цепончка идёт на общие почты — первое письмо строим в логике «передайте коллеге / подскажите, с кем поговорить», а не в лоб на боль ЛПР. Возможен A/B: вариант на общую почту (forward-логика) vs вариант на ЛПР-почту (прямая боль).

## 🔑 Механика CTA — мягкий первый шаг (НЕ сразу демо)

Клиент **согласился** с подходом агентства:
- Цель письма — **первый интерес**, не закрытие на демо.
- CTA №1 = **лид-магнит / консультация / разбор**, что-то ценное за низкий барьер (повышает конверсию в следующий шаг). На сайте уже есть готовый лид-магнит — **бесплатный разбор продаж за 20-30 мин**.
- Дальше: получаем телефон → менеджер квалифицирует → переводит на демо.
- → CTA-лестница цепочки: №1 лёгкий (получить разбор/полезное), не «записаться на демо». Демо — глубже по воронке, в письмах не форсируем.

## Формат кейса в письме («матрица кейса») — как подаём цифры

Стратег задал формат (3 коротких абзаца, не «простыня»):
1. «Видим, что вы работаете в сфере [X]».
2. «У нас по аналитике продаж есть несколько кейсов по вашей нише».

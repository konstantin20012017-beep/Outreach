## Кто заказчик

**«МояКоманда»** (ООО ЦИТ «ФАКТ») — российская облачная **HRM-платформа** для автоматизации HR-процессов и управления талантами. Сайт моякоманда.рф (ismyteam.ru). Старт 2022, разработчик — веб-интегратор «Факт» (Магнитогорск + Россия). Включена в реестр российского ПО.

- Соответствие 152-ФЗ, дата-центр Tier III; размещение Cloud / Private Cloud / On-premise.

## Продукт

**Комплексная HRM-платформа** — весь Employee Journey Map в одном окне: поиск/найм → пребординг → адаптация → удержание/развитие → коммуникации → управление эффективностью → КЭДО → оффбординг. 30 HR-инструментов: рекрутинг (ИИ-оценка, воронка, интеграции hh/superjob/avito), онбординг, LMS, опросы/eNPS, KPI&OKR, 1-1, оценка 360, ИПР, 9-box, геймификация (корп.валюта/магазин), корп.портал/соцсеть, КЭДО, заявки, аналитика текучести, ИИ-ассистент «Тима».

**Ключевое позиционирование (из голосового Кости):** МояКоманда = **комплекс**. Главный угол отстройки — против узкоспециализированных конкурентов (только найм / только опросы) и против «лоскутной» автоматизации (таблицы + Битрикс24 + отдельные сервисы). Один продукт вместо зоопарка инструментов.

**Тарифы (публичны, в письма пока НЕ выносим):** HRM-платформа 10 200 ₽/мес (8 670 за год), демо 14 дней.

## ЦА и сегменты (направление задаёт Костя, голосовое 2026-06-17)

Получатель везде — **ЛПР по HR: HR-менеджер / директор по персоналу (HRD)**. Важная механика: HRD часто **не финальный платящий**, а внутренний адвокат — письмо должно мотивировать его *вынести идею руководству на согласование*. Целевой профиль — малый и средний бизнес (где автоматизация либо лоскутная, либо отсутствует).

### Сегмент 1 — «Прямое обращение к HR/HRD» (объединяет сегменты 1+2 таблицы клиента)
- **Кому:** HR / директор по персоналу в МСБ.
- **Задача письма:** «взломать психологию HRD» — зацепить его личный мотив (разгрузить рутину, выглядеть инициатором улучшений перед руководством), чтобы вышел на связь и захотел протестировать МояКоманду.
- **Боли (идём от проблем):** рутина съедает время HR; HR-данные размазаны по Google-таблицам/Excel; зоопарк узких сервисов не склеивается; нет единой картины по сотруднику.
- **Два сценария отстройки:** (а) автоматизации нет вообще → продать саму идею «зачем»; (б) есть таблицы/Битрикс24/точечные сервисы → показать, что они уступают единой HRM-платформе.
- **УТП-ядро:** комплексность (весь HR в одном окне vs узкие конкуренты), скорость внедрения, российское ПО/152-ФЗ.
- **CTA:** двухшаговый — закрываем на «пришлите номер телефона», чтобы выдать демо-доступ (точную механику демо уточнить у клиента).

### Сегмент 2 — «IT-компании» (отдельный приоритет)
- **Почему приоритет:** у МояКоманды очень много IT-кейсов с цифрами → заходим с социальным доказательством своей отрасли.
- **Угол:** кейсы IT-компаний (SAPE, ММК-Информсервис, SMENA, VisionLabs, Oggetto, ЦИТ/АМЕТУМ, БС-Платформа).
- **Доп-триггер «вакансия»:** открытая вакансия HRD у компании = (1) «власть может меняться» / новый человек ищет инструменты, либо (2) компания активно нанимает → нагрузка на HR растёт, рекрутинг-модуль кстати.
- **CTA:** как С1 (телефон → демо).

### Сегмент 3 — «Универсальный/адаптивный под отрасль»
- **Логика:** один каркас, в который под отрасль получателя подставляется релевантный кейс с сайта (производство → Термо Нова/Сибалюкс; транспорт → Монтранс; стройка/проектирование → Генпро/ЖДС; ритейл/недвижимость → One Moscow; здоровье → Carely).
- Отрасли с готовыми лендингами: IT, Промышленность, Ритейл, Медицина, Транспорт/логистика, Enterprise.

## Сегментация (канон из таблицы клиента) — заменяет ранний черновик
ICP общий: **от 50 сотрудников** (до 50 — не работаем), МС до 1 000 сотр. / выручка 1–3 млрд ₽. Отрасли: IT, промышленность, производство, ритейл, услуги. Гео: РФ/KZ/UZ/РБ. Цикл сделки 1–3 мес.

- **Сегмент 1 — HR (операционный).** Часто «один HR», 50/50 HR или HRD. Работали с узкими решениями / Битрикс24, **не понимают зачем им комплекс/обучение/оценка сейчас**. Боль = подбор/найм/адаптация (операционка): конкуренция за кандидатов, длинный цикл найма, отток на этапах воронки, до 50% времени на рутину, разрозненные данные, текучка на испыт.сроке. Триггеры: укрепление роли HR / сокращение рутины / ускорение найма. → Стадия воронки «Новичок» = месседж-1 «избавьтесь от рутины».
- **Сегмент 2 — HRD (стратегический).** Уже понимает автоматизацию и зачем комплекс. Боль = стратегия: связать HR-метрики с бизнесом, текучка/выгорание, низкий eNPS, формальные performance review, нет ИПР/кадрового резерва. Триггеры: снижение текучки/удержание / доказать ценность HR бизнесу / эффективность без расширения штата. → «Поисковик/Сравниватель» = месседж-2/3/4.
- **Приоритет IT (кросс-сегмент).** Больше всего оцифрованных кейсов. В кейсе решаем задачу компании, в письме добавляем выгоду самому HR («расти HR»). Триггер «вакансия»: массовый найм / вакансия HRD / новые роли / региональная экспансия / плохие отзывы о HR.

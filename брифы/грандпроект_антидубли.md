# Антидубли (ИИ-продукт Гранд Проекта) — спецификация

Источник: https://stroy1c.ru/uslugi/ii-antidubli-v-1s/ (текст прислан клиентом 2026-06-29).

## Суть продукта
ИИ-ассистент для поиска и предотвращения **смысловых (семантических) дублей** в справочниках 1С. Анализирует наименования не по совпадению символов, а **по смыслу**: синонимы, порядок слов, единицы измерения, аббревиатуры, кириллица/латиница, опечатки. Объединяет похожие позиции в группы.

**Месседж клиента:** «Дубли — это не просто беспорядок в базе, это системная утечка денег.» Один материал под разными названиями → остатки расползаются по позициям, единой картины нет, закупщик заказывает то, что уже есть на складе.

## Боль (как сформулировано на лендинге)
Справочник материалов в стройке разрастается до десятков-сотен тысяч позиций. Появляются смысловые дубли. Последствия:
- Искажение остатков (один материал на разных позициях)
- Потеря прозрачности учета (подразделения работают с разными данными)
- Закупка лишних материалов (заявки на то, что уже есть)
- Операционные сбои (ошибки в заказах → срыв сроков)

## 🟢 Цифры эффективности (сравнение методов)
- Ручной просмотр справочника **пропускает 25–30% смысловых дублей** (глаз замыливается на больших базах)
- Встроенный поиск 1С находит **только точные совпадения** по символам, смысловые не видит
- **Антидубли: время на проверку сокращается более чем на 90%, процент пропущенных дублей — не более 3%**
- Режим контроля не даёт новым дублям появляться

## Типы дублей (примеры с лендинга)
- **Полный:** «Арматура A500C Ø12» = «A500C Арматура Ø12» (перестановка слов)
- **Частичный:** «Арматура А500С 12мм» = «Арматура A500C d1.2/см» (разный формат/обозначения)
- **Семантический:** «Лист гипсокартонный влаг. 12.5 мм» = «ГКЛВ 12.5» (разные формулировки, один материал)

## Два режима
- **Аудит:** анализирует весь справочник, отчет по каждой паре с причиной → очистка через MDM-систему
- **Контроль:** проверяет каждую новую позицию перед добавлением, блокирует создание дубля

## Сценарии и выгоды по ролям (с цифрами!)
- **Снабженец/закупки:** захламленный справочник → новая карточка на то, что уже есть → лишняя закупка. **0,5–1% лишних закупок ежемесячно.** Выгода: единая номенклатура, реальные остатки, предупреждение при создании похожей позиции
- **Материальная бухгалтерия:** приход на «Гипсокартон Кнауф 12,5», списание на «ГКЛ влагостойкий Knauf 12,5» → остатки разбиты, пересортица, объяснения аудитору. Выгода: чистые проводки, корректные остатки, прозрачная инвентаризация
- **Производство/прорабы:** заявка из ПТО → снабжение ищет среди похожих → поставка задерживается → простой бригады → срыв сроков. Выгода: заявки стыкуются с реальными остатками, объект по графику
- **CFO/финдир:** бюджет, рентабельность объектов, переговоры с банком — всё на данных 1С. Дубли → завышенная себестоимость, ABC-анализ по задвоенным позициям, план/факт не сходится. **Потенциал снижения себестоимости — 5–15%.** Выгода: достоверная аналитика

## Где работает
Любой справочник: номенклатура (семантические дубли материалов), договоры (дублирующие контракты/версии), характеристики (одинаковые размеры/цвета/марки).

## Особенности
Семантический анализ; гибкие критерии (названия / коды / артикулы / характеристики / ЕИ); контроль при создании; самообучение на данных компании; **интеграция без доработок 1С** (расширение/внешняя обработка, не ломает обновления); **работа в закрытых сетях** (внутри 1С или через личный кабинет — выгрузить справочник, получить отчет без установки); безопасность (обработка в оперативной памяти, без сохранения файлов на сервере); масштабируется на сотни тысяч позиций.

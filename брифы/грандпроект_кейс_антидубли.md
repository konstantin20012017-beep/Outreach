# Антидубли — реальный кейс (стройподрядчик) + актуальные цифры

Источник: статья https://stroy1c.ru/stati/spravochnik-nomenklatury-v-stroitelnoj-kompanii-pochemu-dubli-poyavlyayutsya-bystree-chem-ih-chistyat (опубл. 2026-06-15, теги «1C:Антидубли», «Дубли»). Это ЕДИНСТВЕННЫЙ публичный кейс именно по Антидубли. Анонимный (имя клиента не названо) → в письмах подавать как «у одного крупного стройподрядчика».

## Цифры «анализ 88 компаний» (АКТУАЛЬНЫЕ, клиент подтвердил 2026-06-29)
> Эти 4 цифры — точно использовать (приоритет клиента).
- **5 000 позиций** в справочнике — в среднем с этого размера дубли начинают стоить реальных денег (РАНЕЕ на лендинге было 4 044 → актуально 5 000)
- **до 1%** от суммы закупок уходит впустую каждый месяц из-за дублей
- **42%** компаний признают, что не справляются с дублями даже вручную
- **21%** компаний игнорируют дубли — проблема дорожает каждый месяц

## Метод-цифры (лендинг, продуктовые)
- ручной просмотр пропускает **25-30%** смысловых дублей; Антидубли — **не более 3%**
- время проверки **−90%+**
- потенциал снижения себестоимости **5-15%** (CFO-сценарий, стройка-окрашен)

## КЕЙС: крупный строительный подрядчик (анонимный)
Контекст: работают с ним несколько лет (внедрение 1С + доработка конфигурации). 10 активных объектов одновременно, материалы идут напрямую на объекты. Номенклатуру создают снабжение и ПТО.

**Как появлялись дубли (конкретно):**
- Начинается со сметы: ПТО вносит «профиль потолочный 60×27» — без производителя.
- Снабженец под заявку берет прайс поставщика, а там «Профиль CD 60/27 0.55мм Кнауф». Создает позицию из прайса. **«ПП» и «CD» — один материал (российское и европейское обозначение), но в поиске 1С между ними нет ни одной общей буквы.**
- Другой объект ищет «ПП 60» — «CD 60/27 Кнауф» не находится → создает новую. **Из одной сметной строки вырастают 3-4 записи об одном материале. Без чьей-либо ошибки.**

**Что пытались (и почему не сработало):**
- Папка «Новая номенклатура» (ручная модерация новых позиций) → снабженцы ждут подтверждения, объект стоит; ответственный не успевает; папка за неделю разрасталась до **нескольких сотен позиций**; на нее перестали смотреть.

**Цена проблемы (КОНКРЕТНО — сильный соцпруф):**
- Сложился **отдел материальной бухгалтерии 5-6 человек**, одна из главных задач — порядок в справочнике **каждый день** (не раз в месяц).
- Схлопнуть один дубль = пройти всю цепочку документов (заявка→заказ→поступление→списание) и в каждом подменить позицию. **На один дубль с длинной историей уходил час-полтора. Таких дублей в день находили по несколько штук.**

**Как решили:**
1. Сначала РЕГЛАМЕНТ НСИ (без него инструмент работает вхолостую): формат наименований (существительное→прилагательные→характеристики→производитель), критерии дубля, словари аббревиатур и синонимов (ГОСТ-сокращения), правила для номенклатуры поставщиков + оргчасть (кто создает/модерирует/отвечает).
2. Потом продукт Антидубли (ИИ, ищет по смыслу как человек со знанием правил). **3 режима:** Контроль при создании (в реальном времени, показывает «похожее уже есть» до сохранения) / Аудит раз в месяц (прогон всего справочника) / Устранение (схлопывание с учетом MDM/закрытых периодов).

**Результат (⚠️ КАЧЕСТВЕННЫЙ, без %):** «заказчик сейчас тратит на ручное сопоставление дублей заметно меньше времени — часть работы делает система». Опубликованного % сокращения / суммы экономии в кейсе НЕТ.

**4 вывода кейса (хорошие тезисы для писем):**
1. Дубли — не из-за халтуры, а потому что система не помогает видеть, что уже есть под другим названием.
2. Перед автоматизацией — регламент НСИ, иначе инструмент вхолостую.
3. Нужны процессы (контроль при создании + регулярный аудит) и ответственные.
4. Смысловой поиск дублей человек физически не тянет в масштабе тысяч позиций.

## Вывод для писем (честно)
- **Твердые ЦИФРЫ = только агрегат «88 компаний» (5000/1%/42%/21%) + метод (25-30% vs 3%, −90%, 5-15%).** Их и использовать.
- **Кейс стройподрядчика = сильный КОНКРЕТНЫЙ соцпруф** (отдел 5-6 человек чистил каждый день, 1-1,5 часа на дубль, пример ПП/CD), но БЕЗ итогового %. Подавать как живую иллюстрацию, не как «-X% экономии».
- Именного клиента с опубликованным до/после нет → если нужен hard-ROI на клиента, запросить у заказчика.
- Кейс — стройка (родной для С1). Для С2/С3 — либо честная кросс-атрибуция «у стройподрядчика», либо запросить кейс по опту/промышленности.

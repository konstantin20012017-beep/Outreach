Что продаём и общая механика (единая для 3 цепочек)

  • Продукт — SEO-продвижение, но в письме SEO напрямую не продаём.
  • Задача цепочки — закрыть на ответ «интересно» → менеджер просит телефон, хвалит, ведёт на встречу, и только потом отдаёт ценность.
  • Механика ответа: получатели ничего ответным письмом не высылают — просят контакт и ведут на звонок. CTA письма = получить «интересно»/контакт.
  • Приманка во всех цепочках — бесплатный аудит (можно закрывать на разное: аудит / план действий по сайту / стратегия продвижения).

Цепочка 1 — Персонализированный аудит (производства/заводы)

Угол: айсбрекер = персональный SEO-показатель по сайту получателя.

  • Механика айсбрекера: вычисляем производственные компании по сайту → инструментами снимаем SEO-показатели → добавляем уникальный параграф под конкретный сайт. Письмо мимикрирует под ручную проверку.
  • ЦА/ЛПР: производства, заводы. Маленький завод → руководитель/собственник; большой → директор по маркетингу / маркетолог. Добавляем «передайте коллегам».
  • Цель (3 письма): продать именно аудит, не SEO.
  • CTA: «интересно?» → бесплатный аудит.
Каркас №1 (логика стратега): «[приветствие]. Проверили ваш сайт в Яндексе/Гугле (показатель/инструмент согласует Евгений) и увидели вот такие уникальные параметры (1 строка — персональный параметр). Похоже, по SEO теряете клиентов. Чтобы обогнать конкурентов — проведём аудит и всё бесплатно расскажем. Интересно?»
Персональный SEO-параметр = плейсхолдер (оператор подставляет реальные данные). Не выдумывать показатели. Формулировку «что именно проверили и какой параметр» согласует Евгений — открытое поле.

Цепочка 2 — Триггер «вакансии» (любые отрасли)

Угол: видим открытую вакансию (сеошник/маркетолог) → предлагаем рассмотреть аутсорс как альтернативу найму.

  • Триггер: открытая вакансия. Любые вакансии, любые отрасли.
  • 🔑 Ключевой аргумент: насмотренность и экспертиза агентства — через него проходит больше клиентов, чем видит штатный спец; больше кейсов и опыта.
  • Легко начать / легко остановить (vs наём-увольнение).
  • По деньгам может быть доступнее; работает целая команда по цене одного специалиста.
  • Безопасность: подписываем NDA — важно для заводов.
  • CTA: «интересно?» → вышлем аудит / план / стратегию (в каждом письме можно закрывать на разное); по факту ценность отдаётся на звонке.
Разводка C1/C2 по бизнес-логике: C1 = «нашли проблему на вашем сайте», C2 = «вы решаете задачу наймом — есть способ выгоднее». По триггеру и боли, не по лексике.

Цепочка 3 — Матрица кейсов (под нишу/кейс)

Угол: «в вашей нише у нас уже есть результат» — пишем под нишу получателя на базе похожего кейса.

  • Механика: берём кейс → находим получателей в той же нише → «к нам обратилась похожая компания, была проблема, решили, вот результаты — хотите так же?».
  • Единственный пример из инструкции: кейс Falcon — интернет-магазин свето-тюнинга. Пишем под «свето-тюнинг» или «интернет-магазин».
  • Подача: упор на цифры/результат, а не на процесс. Год не указывать.
  • CTA: «напишите интересно — свяжемся» + упомянуть бесплатный аудит.
Цифры кейсов — только из реальных кейсов клиента (сайт/документы), которых пока нет. Falcon без подтверждённых цифр в цепочку не идёт. C3 = матрица: по примеру цепочки на ключевой кейс/нишу.

Формат

  • Стандарт: 3 письма в каждой цепочке, тема в каждом письме (одинаковая по сегменту, с {Компания}), приветствия/подписи — плейсхолдеры [приветствие] / [подпись].

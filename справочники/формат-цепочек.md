# Формат HTML-цепочек — справочник

Технические правила вёрстки HTML-файла цепочки. Применяются при создании/правке любого файла в `цепочки/*.html`. Подгружается при работе над цепочкой через skill `аутрич-цепочка`.

## Базовые правила формата писем

Правила «тему/приветствие/подпись не писать» и «без ё, тире `–`, дефис `-` для цифр» — красные линии в CLAUDE.md, здесь не повторяю.

Структурные правила HTML-цепочки:
- **3 письма в цепочке** — всегда ровно 3, не больше и не меньше (исключение — явная просьба клиента).
- **Нумерация писем** — «Письмо №1», «Письмо №2», «Письмо №3» (не капсом, с заглавной).
- **Тема — в каждом письме** (правило с 2026-05-29). Отдельной строкой `<p><b>Тема:</b> ...</p>` в начале блока письма, перед `[приветствие]`. **Тема одинаковая во всех 3 письмах** (склейка в одну цепочку в почтовике), с переменной `{Компания}`. Тема общая по сегменту, не под роль письма. Приветствия/подписи — по-прежнему только плейсхолдеры.

## HTML-стили (под Google Docs paste)

- **Сохранение** — всегда в папку `цепочки/<проект>_<сегмент>_v<N>.html`.
- **Кнопка копирования** для каждого письма отдельно + кнопка «Копировать все письма» в начале.
- **Межстрочный интервал** — line-height: 1.15.
- **Шрифт** — Arial, размер 11pt для основного текста.
- **Заголовки** (названия сегментов, номера писем) — оформлять как Heading 2 Google Docs:
  ```html
  <h2 style="font-family: Arial; font-size: 18pt; font-weight: bold;">Письмо №1</h2>
  ```
  Это нужно, чтобы при копировании из HTML в Google Документы форматирование подхватилось автоматически (заголовки — Heading 2, тело — обычный Arial 11pt).
- **Inline styles на каждом блоке** — без явного `font-family: Arial; font-size: 11pt;` в каждом `<p>` Google Docs при вставке может поломать шрифт.

## Маркер PRE-FLIGHT

После прохождения чеклиста по skill'у `аутрич-цепочка` вставить в HTML:
```html
<!-- PREFLIGHT-OK -->
```
Например, сразу после `<head>`. Без этого маркера hook блокирует Write.

## Структура HTML

Сверху до писем:
1. **Заголовок** — название цепочки и версия.
2. **Кнопка «Копировать все письма»**.
3. **Угол цепочки** — какой заход выбран и почему (1-2 абзаца).
4. **Warning-карточки** — если что-то убрано/изменено/недостаёт (см. ниже).
5. **Таблица разводки сегментов** (если их 2+) — по 4 осям: категории / триггер / боль / УТП.
6. **Таблица распределения по письмам** — строки №1/№2/№3, колонки Боль / Регалии / УТП.
7. **Источники** — точные цитаты из брифа/кейсов/созвона/лендинга по каждой цифре.
8. **Письма** — каждое в `<div id="...">` с кнопкой копирования.

## Кнопка копирования (Clipboard API, чистая копия в Google Docs)

Клонировать узел и вырезать `.copy-btn` / `.seg-divider` перед записью в буфер. Записывать оба формата:
```javascript
async function copyBlock(id, btn) {
  const src = document.getElementById(id);
  const clone = src.cloneNode(true);
  clone.querySelectorAll('.copy-btn, .seg-divider').forEach(el => el.remove());
  const html = clone.innerHTML;
  const text = clone.innerText;
  const blob = new Blob([html], { type: 'text/html' });
  const textBlob = new Blob([text], { type: 'text/plain' });
  await navigator.clipboard.write([new ClipboardItem({ 'text/html': blob, 'text/plain': textBlob })]);
  btn.textContent = 'Скопировано';
  setTimeout(() => btn.textContent = 'Копировать', 1500);
}
```

## Warning-карточки

Когда конкретики не хватает или что-то умышленно убрано — отдельный блок с жёлтым фоном перед таблицами:
```html
<div class="warn">
  <b>Что убрал:</b> ...
  <br><br>
  <b>Что запросить у клиента:</b> ...
</div>
```
Стиль:
```css
.warn { background: #fff3cd; border-left: 4px solid #f0ad4e; padding: 12px 16px;
        border-radius: 6px; margin: 10px 0; font-size: 13px; }
```

## Открывать готовые файлы

Всегда открывать готовый HTML в Google Chrome сразу после Write:
```bash
open -a "Google Chrome" /путь/к/файлу.html
```

Брифы — тоже в HTML (не только markdown), чтобы читались в браузере.

## Pre-flight grep'ы (после Write)

```bash
f=имя_файла.html
grep -nE "Здравствуйте|Добрый день|доброгоВремениСуток|С уважением|^Подпись" "$f"
grep -nE "не [А-Яа-я]+.*, а |, а не |вместо .* —" "$f"
grep -nE "ё|Ё" "$f"
grep -nE "[0-9]–[0-9]" "$f"
grep -nE "—" "$f"
grep -niE "крупны[ех] (компани|игрок|бренд)|больши[ех] (компани|бренд)|известны[ех] бренд|ведущи[ех] компани|есть бренды, которые|среди заказчиков есть" "$f"
```
Первые 5 должны возвращать пусто. Что нашлось — переписать.

**6-й grep — размытый соцпруф** (правило [[feedback_concrete_brands_not_vague]]): ловит обобщённые формулировки про клиентов («крупные компании», «среди заказчиков есть…», «известные бренды»). Найдено — заменить на конкретные публичные имена из портфолио (разнести по письмам) или вынести в open field. Это семантическая ловушка, не форма: проверять глазами каждое предложение про клиентов, даже если grep пуст (фразу можно построить иначе).
